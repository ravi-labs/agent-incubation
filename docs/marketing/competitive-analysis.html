Agent Framework Competitive Analysis

agent-foundry vs. the field  ·  Q1 2026 Research Preview

Frameworks in scope
agent-foundry (us)
Pre-execution typed policy gate
Manifest-declared effect scope, hard-deny layer, domain taxonomies (Financial / Healthcare / Legal), built-in audit JSONL + HTML report, Lambda + local deploy.
Microsoft Agent Framework
AutoGen + Semantic Kernel unified
RC Feb 2026. Graph-based workflows, multi-pattern orchestration (sequential, concurrent, group-chat, handoff, Magentic), A2A / AG-UI / MCP interop. Python + .NET.
MS Agent Governance Toolkit
Cross-framework governance add-on
Open-source. Pre-execution capability model with allowed/denied tools, zero-trust agent identity (Ed25519 / SPIFFE), <0.1 ms overhead. Integrates with 12+ frameworks. Covers OWASP Agentic Top 10.
LangGraph
Graph-based stateful orchestration
Explicit DAG/state machine for agents. Durable execution, checkpointing, strong human-in-the-loop. LangSmith for observability. Backed by LangChain Inc.
CrewAI
Role-based agent teams
Intuitive crew/role/task model. Fastest setup, growing A2A support, commercial enterprise tier. Can model approval flows as agent roles. Most GitHub stars.
Bedrock AgentCore (AWS)
Managed cloud agent runtime
Announced re:Invent 2025. Real-time tool-call interception, Cedar policy language (natural language → policy), SOC 2 / HIPAA compliance. AWS IAM integration.
NeMo Guardrails (NVIDIA)
Rule-driven output supervision
Colang configuration language for topic rails, output filtering, and tool-access rules. Post-hoc and inline guardrail patterns. Open-source, model-agnostic.
OpenAI Agents SDK
Tool use + handoffs + tracing
Open-source Python/TS SDK. Tool use, agent handoffs, built-in tracing. Provider-agnostic via AGENTS.md spec. Guardrails hooks available but no typed taxonomy.

Strategic verdict

Where agent-foundry wins unconditionally

Typed, domain-specific effect taxonomy — No competitor ships a prebuilt FinancialEffect, HealthcareEffect, or LegalEffect enum. Bedrock AgentCore uses Cedar (generic resource strings). The MS Governance Toolkit uses allow/deny tool lists. We have 100+ semantically-typed effects that map directly to compliance vocabulary (ERISA, HIPAA, UPL). This is a genuine moat.

Policy regression testing — Writing a test that says "this agent MUST DENY FinancialEffect.TRADE_EXECUTE" is only possible with a typed taxonomy. No other framework can do this. LangSmith evals check output quality; we check governance correctness.

Offline audit report — Self-contained HTML with zero SaaS dependency is a real differentiator for regulated industries (banking, healthcare, legal) that cannot send audit data to a third-party cloud.

Where we need to close the gap

Orchestration — MAF and LangGraph are materially ahead. Adding a basic multi-agent router (handoff + sequential graph) would keep enterprise prospects from having to bolt on LangGraph just for orchestration.

A2A / MCP interop — The ecosystem is converging on these standards. Shipping MCP-native tool dispatch would let agent-foundry tools be discovered and called by MAF or CrewAI agents — turning governance into a cross-ecosystem feature rather than an island.

Managed cloud runtime option — Bedrock AgentCore's fully managed model removes an ops burden. A lightweight Foundry-as-a-service wrapper (even just a Docker Compose + Terraform module) would lower the adoption barrier for non-platform teams.
